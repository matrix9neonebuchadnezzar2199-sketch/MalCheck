Static analysis JSON schema (analysis.json)

Last updated: 2026-06-04. Produced by scripts/ghidra/auto_analyze.py inside the ghidra-headless image. Consumed as phase3_static.analysis_json after mau/static_normalize.py.

Top-level fields

FieldTypeDescription
file_namestringProgram name in Ghidra
architecturestringProcessor / language ID
compilerstringCompiler ID when known
entry_pointsstring[]Entry addresses
functionsobject[]Per-function analysis (see below)
stringsobject[]{ address, value }
importsobject[]{ library, function, address }
exportsobject[]{ address, kind }
suspicious_apisobject[]Roll-up of risky imports / calls
truncatedbooleanTrue if export hit internal limits
call_graphobject{ nodes[], edges[], truncated? }

Per-function (functions[])

  • name, address, size, decompiled_c
  • cfg — basic blocks and edges (bounded)
  • xrefs — callers / callees
  • decompile_insights — heuristic signals on decompiler text
  • line_address_map — decompiler line → address

Report mapping

phase3_static: {
  "engine": "ghidra_headless",
  "status": "completed" | "failed",
  "analysis_json": { ... },
  "summary": {
    "function_count": 0,
    "suspicious_api_count": 0,
    "truncated": false,
    "suspicious_apis_top": []
  },
  "decompiled_c": "...",
  "functions": { ... },
  "metadata": { ... }
}

Legacy 3-pass output (decompiled.c, functions.json, metadata.json) remains available when MAU_GHIDRA_LEGACY=1 is set on the container.
